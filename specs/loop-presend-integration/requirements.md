# 循环模式 + 预发送集成 - 需求文档

## 1. 功能概述

将现有的循环模式（Loop Mode）与预发送队列系统（Pre-Send Queue）深度集成。弃用原有的 `/loop` 斜杠命令拼接方式，改为由预发送队列自行管理循环逻辑，支持循环次数、间隔时间设置，并允许循环消息与普通预发送消息混合排队。

## 2. 用户故事

### US-1：设置循环次数和间隔
作为星迹的CC用户，我希望在开启循环模式后，能在预发送栏右侧看到次数和间隔输入框，以便精确控制循环行为。

- 次数输入框：数字类型，placeholder = "∞"，空值表示无限次
- 间隔输入框：数字类型，单位"分钟"，默认值 10
- 两个输入框仅在循环模式开启时显示

### US-2：循环内容自动排队
作为星迹的CC用户，我希望输入内容按 Enter 后，该内容自动进入预发送队列并标记为循环项，AI 每完成一轮就自动将该内容重新加入队尾，直到达到循环次数。

### US-3：混合排队
作为星迹的CC用户，我希望在循环进行中，可以继续输入其他消息并排队，这些消息按顺序排在当前轮次和下一轮之间。

- 场景：正在执行第 3 次"你好"，用户追加"早上好" → "早上好"排在第 3 次和第 4 次之间
- 预发送模式开启时：消息进队列排队
- 预发送模式关闭时：消息正常立即发送（和原有 loop 行为一致，但不拼 `/loop` 命令）

### US-4：普通消息不等待循环间隔
作为星迹的CC用户，我希望队列中的普通（非循环）消息在 AI 空闲时立即发送，不需要等待循环间隔。

### US-5：取消循环
作为星迹的CC用户，我希望能方便地取消循环：
- 点击发送按钮（忙碌时）：中断当前任务，清空预发送队列
- 按 Esc：取消当前正在等待的循环，保留队列中的其他普通消息
- 切换出循环模式：停止循环自增，保留队列中已有消息

### US-6：循环状态持久化
作为星迹的CC用户，我希望刷新页面后循环状态能恢复（通过 localStorage 持久化）。

## 3. 验收标准

### AC-1：循环模式 UI
```
Given  用户切换到循环模式
When   循环模式激活
Then   预发送预览栏右侧显示两个输入框：
       - 次数输入框（placeholder "∞"，默认空）
       - 间隔输入框（默认 10，单位"分钟"）
```

### AC-2：循环发送流程
```
Given  循环模式开启，次数=3，间隔=1
When   用户输入"你好"并按 Enter
Then   "你好"进入预发送队列，显示循环标识
And    AI 空闲时立即发送第 1 次"你好"
And    AI 完成后等待 1 分钟，自动发送第 2 次"你好"
And    AI 完成后等待 1 分钟，自动发送第 3 次"你好"
And    3 次完成后循环停止，队列清空
```

### AC-3：无限循环
```
Given  循环模式开启，次数=空（无限）
When   用户输入"你好"并按 Enter
Then   循环持续发送"你好"，永不停止
And    用户点击发送按钮或按 Esc 可停止
```

### AC-4：混合排队顺序
```
Given  循环模式开启，正在执行第 3 次"你好"
When   用户追加"早上好"
Then   "早上好"排在第 3 次"你好"之后、第 4 次"你好"之前
And    第 3 次完成后立即发送"早上好"（不等待循环间隔）
And    "早上好"发送完成后等待间隔时间，再发送第 4 次"你好"
```

### AC-5：预发送关闭时的行为
```
Given  循环模式开启，但预发送模式关闭
When   用户输入"你好"并按 Enter
Then   消息正常发送，不进预发送队列
And    不拼接 /loop 斜杠命令
```

### AC-6：中断清理
```
Given  循环模式运行中
When   用户点击发送按钮
Then   AI 被中断，预发送队列完全清空，循环停止
```

## 4. 非功能性需求

- 性能：轮询间隔保持 300ms，不因循环逻辑增加额外开销
- 兼容性：不影响现有的非循环预发送功能
- 持久化：循环状态随预发送队列一起存入 localStorage
- 代码质量：修改集中在 `webview/index.js`，不引入新依赖

## 5. 影响范围

### 直接修改
- `extension/webview/index.js`：
  - `send()` 方法（行 143053）：移除 `/loop` 前缀拼接，改为调用预发送队列
  - `_doPoll()` 函数（行 207464）：增加循环自增逻辑
  - `_createPreview()` 函数（行 207308）：增加次数/间隔输入框 UI
  - `_clearPS()` 函数（行 207300）：增加循环状态清理
  - `__preSendState` 数据结构：增加 `loop` 字段
  - 普通入队逻辑（行 207514-207680）：支持循环消息入队
  - 发送按钮中断逻辑（行 206229）：增加循环状态清理

### 间接影响
- `window.__preSendState` 的数据结构变化影响所有读写该状态的代码
- localStorage 格式变化需要向后兼容（旧数据不含 loop 字段时按普通队列处理）

### 不受影响
- `extension/extension.js`：不需要修改，preSend 配置读写逻辑不变
- `extension/package.json`：不需要新增配置项
- 权限模式切换（`setPermissionMode`）：逻辑不变，仍支持 "loop" 模式
