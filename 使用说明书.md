# 星迹的CC✨ 使用说明书

> 版本：2.1.95 | 基于 Claude Code 官方版 2.1.95 深度定制

---

## 一、简介

**星迹的CC✨** 是将 Anthropic 官方 Claude Code 完整汉化并增强的 VS Code 扩展。它让你在编辑器内直接与 Claude AI 对话，自动读写代码、运行终端命令、管理 Git，并支持子代理、MCP 等高级 Agent 功能。

### 核心亮点

| 特性 | 说明 |
|---|---|
| **全自动 YOLO 模式** | 跳过所有权限确认，Claude 直接执行全部操作 |
| **预发送 (Pre-Send)** | AI 运行时排队发送消息，不打断当前生成 |
| **全面汉化** | 界面、提示、状态信息均已翻译为中文 |
| **自定义音频反馈** | 任务完成/报错/提问时播放自定义音频 |
| **多权限模式** | 支持 bypassPermissions / acceptEdits / plan / default 四种模式 |
| **灵活界面** | 支持侧边栏、新标签页、终端三种打开方式 |

## 二、安装教程

### 2.1 安装 VSIX 包

1. 打开 VS Code
2. 按 `Ctrl+Shift+P` 打开命令面板
3. 输入 `Extensions: Install from VSIX`
4. 选择 `xingji.claude-code-2.1.95.vsix` 文件
5. 等待安装完成，点击「Reload」重新加载窗口

### 2.2 系统要求

- **VS Code**：1.94.0 或更高版本
- **操作系统**：Windows / macOS / Linux
- **Claude 账号**：需 Claude Pro / Max / Team / Enterprise 订阅，或按用量计费

## 三、快速上手

### 3.1 打开插件

安装完成后，可以通过以下方式打开：

- **快捷键**：`Ctrl+Escape` 在面板中打开，`Ctrl+Shift+Escape` 在新标签页中打开
- **活动栏图标**：点击左侧活动栏的 Claude 图标
- **命令面板**：`Ctrl+Shift+P` → 搜索「星迹的CC」

### 3.2 第一次对话

1. 打开插件后在底部输入框输入你的需求
2. 按 `Enter` 发送消息
3. Claude 会开始分析你的代码库并回复

**小技巧**：使用 `@` 符号可以快速引用文件、文件夹或 Git 提交，为 Claude 提供上下文。

### 3.3 界面说明

```
┌─────────────────────────────────┐
│  ← 返回    历史对话  ✨ 新建对话   │  ← 顶部工具栏
├─────────────────────────────────┤
│                                 │
│   Claude 的回复区域              │  ← 对话显示区域
│   (支持代码高亮、Diff 预览等)      │
│                                 │
├─────────────────────────────────┤
│ > 输入你的消息...          [发送] │  ← 底部输入框
│ [权限模式选择器] [预发送开关]      │  ← 功能控制

```

## 四、权限模式

插件提供四种权限模式，可在界面左下角切换：

| 模式 | 说明 | 适用场景 |
|---|---|---|
| **🚀 YOLO (bypassPermissions)** | 跳过所有权限确认，全自动执行 | 完全信任 AI，追求效率 |
| **📝 接受编辑 (acceptEdits)** | 只确认文件编辑操作 | 半自动，审核代码修改 |
| **📋 计划 (plan)** | AI 仅做规划，不执行操作 | 先讨论方案再动手 |
| **🛡️ 默认 (default)** | 所有操作需确认 | 最安全，适合新手 |

### YOLO 模式特别说明

YOLO 模式下 Claude 会自动执行所有操作（读写文件、运行命令、Git 操作等）。如需在 YOLO 模式下仍保留提问的交互能力，可开启设置中的 `Ask User Question` 选项。

## 五、预发送功能 (Pre-Send)

预发送功能类似 Codex 的 Steer，让你在 AI 还在生成回复时就能预先排队发送下一条消息。

- **开启方式**：点击输入框左侧的预发送开关，或在设置中启用 `Claude Code › Pre Send`
- **使用方式**：AI 运行中按 `Enter`，消息不会打断当前生成，而是进入排队队列，等 AI 完成后自动发送
- **视觉提示**：排队中的消息会有醒目的视觉标识

## 六、自定义音频反馈

插件支持在任务完成、报错或 AI 提问时播放自定义音频（替代系统默认 beep 音）。

### 启用方法

在 VS Code 设置中搜索 `claudeCode.audioFeedback`，开启 `Enabled` 选项。

### 自定义音频文件

| 设置项 | 默认音频 | 触发场景 |
|---|---|---|
| `Completion Sound Path` | `resources/audio/主人，任务已完成，请您查收~.mp3` | 任务完成 |
| `Error Sound Path` | `resources/audio/糟糕，主人，好像遇到报错啦~快来救救我！.mp3` | 任务报错 |
| `Question Sound Path` | `resources/audio/主人，我有个问题想问你。.mp3` | AI 提问 |

你可以替换为自己的音频文件（支持 mp3 / wav / ogg / webm 格式）。

## 九、设置选项一览

在 VS Code 设置中搜索「星迹的CC」查看全部设置项。以下是常用设置：

| 设置项 | 默认值 | 说明 |
|---|---|---|
| `Initial Permission Mode` | bypassPermissions | 新对话的默认权限模式 |
| `Pre Send` | 关闭 | 启用消息预发送队列 |
| `Ask User Question` | 开启 | YOLO 模式下弹出提问选项 |
| `Task Completion Notification` | 开启 | 任务完成时弹窗通知 |
| `Task Completion Sound` | 开启 | 任务完成时播放提示音 |
| `Use Ctrl+Enter To Send` | 关闭 | 改为 Ctrl+Enter 发送，Enter 换行 |
| `Autosave` | 开启 | Claude 操作前自动保存文件 |
| `Respect Gitignore` | 开启 | 搜索时遵守 .gitignore 规则 |
| `Preferred Location` | panel | 默认打开位置（侧边栏/面板） |
| `Audio Feedback` | 关闭 | 自定义音频反馈 |
| `Hide Onboarding` | 关闭 | 隐藏新手引导清单 |

## 十、改造须知（与原版区别）

> 本节详列本扩展相对于 Anthropic 官方 Claude Code VS Code 扩展（2.1.95）的全部改动。

### 10.1 品牌与界面

| 改动点 | 原版 | 改造后 |
|---|---|---|
| 扩展名称 | Claude Code | 星迹的CC✨ |
| 界面语言 | 英文 | 全面汉化（所有命令、设置、提示文本） |
| 活动栏标识 | Claude 图标 | Claude 图标 + ✨ |

### 10.2 权限模式改动

| 改动点 | 原版 | 改造后 |
|---|---|---|
| 默认权限模式 | `default`（每次操作确认） | `bypassPermissions`（YOLO 全自动） |
| 可选模式数量 | 3 种（default / acceptEdits / plan） | 4 种（含 bypassPermissions） |
| YOLO 陷阱保护 | 无 | 内置安全陷阱，拦截高风险命令 |
| AskUserQuestion 控制 | 无独立开关 | YOLO 模式下可配置为手动选择 |

### 10.3 新增功能

| 功能 | 说明 |
|---|---|
| **预发送 (Pre-Send)** | AI 运行时按 Enter 排入队列，等待 AI 完成后自动发送，不中断当前生成。类似 Codex Steer 的预发送体验 |
| **自定义音频反馈** | 支持 mp3/wav/ogg/webm 格式，分别为任务完成、报错、AI 提问设置不同音频提示 |
| **预发送队列视觉提示** | 排队中的消息有醒目的颜色标识，队列数量实时显示 |

### 10.4 配置默认值改动

| 设置项 | 原版默认值 | 本版默认值 |
|---|---|---|
| `initialPermissionMode` | `"default"` | `"bypassPermissions"` |
| `audioFeedback.enabled` | 无此配置项 | `false`（可选开启） |
| `audioFeedback.completionSoundPath` | 无 | 自定义音频路径 |
| `audioFeedback.errorSoundPath` | 无 | 自定义音频路径 |
| `audioFeedback.questionSoundPath` | 无 | 自定义音频路径 |
| `preSend` | 无此配置项 | `false`（可选开启） |
| `askUserQuestion` | 无此配置项 | `true` |

### 10.5 技术实现说明

- **修改方式**：基于官方 2.1.95 版本源码手工打补丁（非自动化 diff/merge）
- **修改文件**：`extension/extension.js`、`extension/package.json`、`extension/webview/index.js`、`extension/webview/index.css`
- **新增文件**：`extension/resources/audio/` 目录下的 3 个自定义音频文件
- **扩展 ID**：`xingji.xingjiclaudecode`（与原版 `anthropic.claude-code` 独立，可共存）
- **版本策略**：官方发布新版本后，其代码经过再次混淆（minify），无法直接 diff 对比升级。需要将官方新功能手工回移植到本版的对应位置，而非整体替换

### 10.6 已知限制

- 非 Anthropic 官方发布版，不受官方技术支持
- 官方新功能需手工回移植，无法自动化同步
- 部分高级功能（自定义斜杠命令、MCP 服务器）需通过 CLI 配置

## 十一、常见问题

**Q: 安装后无法使用？**
A: 确保 VS Code 版本 ≥ 1.94.0，安装后需要 Reload Window。如仍无法使用，检查是否需要登录 Claude 账号。

**Q: YOLO 模式执行了我不想要的操作怎么办？**
A: 使用 VS Code 的 `Ctrl+Z` 撤销文件修改，Git 操作可通过 `git reflog` 找回。建议先在 plan 模式预览 AI 的计划，再切换回 YOLO。

**Q: 如何切换回终端风格？**
A: 命令面板搜索「星迹的CC: 在终端中打开」，或在设置中开启 `Use Terminal`。

**Q: 预发送的消息在哪里查看？**
A: 输入框中排队的消息会有橙色/金色视觉标识，AI 完成后会自动发送。

**Q: 如何更新插件？**
A: 命令面板搜索「星迹的CC: 更新扩展」，或重新安装新版 VSIX 包。

**Q: 插件体积为什么这么大？**
A: VSIX 包约 75MB，主要因为内置了 Claude CLI 原生二进制文件 (claude.exe)，确保开箱即用。

**Q: 可以和官方版同时安装吗？**
A: 可以。本扩展 ID 为 `xingji.xingjiclaudecode`，与原版 `anthropic.claude-code` 相互独立，两个扩展可共存。
